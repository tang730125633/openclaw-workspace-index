> 此模板由 Tang 于 2026-04-08 确认通过，所有6h大报必须遵循此格式。

## 格式规范

```
📊 **戴总企业服务 | {{日期}} {{时间}} 综合汇报**

**━━ 一、业务进展 ━━**

🟠 **早报系统** — {{进度状态}} {{完成%}}%
{{一句话说明当前状态}}

🔵 **知识库系统** — {{进度状态}} {{完成%}}%
{{一句话说明当前状态}}

🟣 **人格系统** — {{进度状态}} {{完成%}}%
{{一句话说明当前状态}}

🟤 **图像生成系统** — {{进度状态}} {{完成%}}%
{{一句话说明当前状态}}

📌 **过去 6 小时工作总结**
• {{要点1}}
• {{要点2}}
• ...

📎 [业务进展表]({{Base链接}}) | [任务队列]({{任务队列链接}}) | [轮询日志]({{轮询日志链接}})

**━━ 二、系统技术状态 ━━**

A网关{{✅/⚠️}} B飞书CLI{{✅/⚠️}} C渠道{{✅/⚠️}} D知识库{{✅/⚠️}} E脚本{{✅/⚠️}} F索引{{✅/⚠️}} — **{{全绿/有异常}}**

**轮询健康度（今日）**
🤖 小婷：轮询 N 次，执行 N 次任务 — 任务 ✅N/❌N/⏳N
🤖 小琳：轮询 N 次，执行 N 次任务 — 任务 ✅N/❌N/⏳N
📊 双端合计 N 次轮询，N 次有效执行，系统心跳{{正常/异常}}

下次汇报：{{下一个 0/6/12/18 点}}
```

## 关键规则
1. **只发一条消息**，不要追加执行摘要
2. **必须带飞书多维表格链接**
3. **去掉 ASCII 框线图**，用 emoji + 粗体
4. **轮询健康度归入第二段**（系统技术状态），不单独成段
5. **6小时工作总结**基于表格数据 + 近期对话内容，3-5 个要点
6. **完成后推送 GitHub**：reports/{{YYYY-MM-DD}}-{{HHmm}}-综合汇报.md
